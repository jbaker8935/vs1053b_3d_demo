VS1053b Geometry Kernel User Guide

This guide documents the geometry kernel host API, memory-model assumptions, and render-flow integration needed to draw wireframe objects and scenes on K2 bitmap layers.

1. Introduction

The geometry kernel runs on the VS1053b DSP and acts as a 3D coprocessor for the host CPU. The host configures models, camera/object transforms, and projection, then triggers the DSP and reads projected edge results for rasterization by the K2's hardware line drawing engine.

Core capabilities include:

  • 3d object loading into kernel memory (8 save slots)
  • 3d scaling, translation and 3-axis rotation
  • vertex screen projection and edge clipping
  • hidden-line removal using face normals
  • near/far depth object edge coloring
  • camera translation and rotation
  • scene definition of up to 8 objects
  • scene rendering with or without AABB occlusion

The VS1053b supports user application though its plugin interface, which allows custom code to run on the DSP and access shared RAM buffers. The geometry kernel is implemented as a plugin that occupies the DSP and exposes a C API to the host for SCI-based control and data exchange. VS1053b audio processing should be disabled when using the plugin.

2. Operation

Host/DSP Communication Model

  • Host writes parameters and geometry into VS1053b shared RAM regions using the K2's SCI interface to the VS1053b.
  • Host triggers processing by writing VGK_TRIGGER_MAGIC (0xCAFE) to SCI_AICTRL0. The VS1053b detects the write to that register and calls the plugin dispatch address
  • The geometry kernel executes transform and projection operations and then updates the status memory location with VGK_STATUS (idle, busy, done, error codes).
  • Host polls the status using SCI and then reads output edge/screen buffers to pass to the hardware line drawing engine.

Pipeline Stages

  1. Load model into one of 8 save slots with vgk_model_slot_init or part-wise slot init calls.
  2. Set camera/object parameters using Euler angles, scale, and translation.
  3. Trigger DSP with vgk_trigger.
  4. DSP computes trig terms using an internal lookup table, builds the transform matrix and transforms vertices.
  5. DSP performs perspective projection and clipping.
  6. Optional hidden-line test uses face normals + edge-face adjacency.
  7. DSP writes output packed edges, flags, and clip vertices.
  8. Host reads output and draws lines onto the target bitmap layer.

Numeric Representation

  • Angles: 8-bit index into a 256-entry sine table ( 0..255 == 0..2*pi).
  • Trig and matrix terms are Q14 fixed-point.
  • Object scale is Q7 (128 means 1.0x).
  • Coordinates are int16_t model/world/view values.
  • World coordinate range: -16384..16383

Depth Classification

Each visible edge can be flagged as near or far (VGK_EDGE_NEAR) by the kernel and the host can use different palette colors. By default the color pair used by the host comes from Model3D.object_color (high byte = far, low byte = near). Near/far coloring is optional and can be disabled for better performance.

3. Configuration Options

Projection Control

  • vgk_projection_params_init(int16_t focal, int16_t half_w, int16_t half_h, int16_t near_z)
  • vgk_projection_enable(void)
  • vgk_projection_disable(void)

focal, half_w, half_h, and near_z directly shape perspective and clipping.

Hidden-Line Removal

  • vgk_hidden_line_enable(void)
  • vgk_hidden_line_disable(void)

Hidden-line mode requires valid face normal and edge-face data in the loaded slot. If a model has no face data then objects will be rendered as wireframe.

Depth Coloring Controls

  • Model3D.object_color: packed 16-bit near/far palette indices.
  • Model3D.edge_color/edge_color_count: optional per-edge color overrides. FUTURE: renderer does not currently implement object defined edge coloring
  • vgk_no_near_far_coloring global: force all edges to near color when true.

Yield Hook During Wait Loops

  • vgk_yield_cb_set(void (*cb)(void))
  • vgk_yield(void)

Use the callback to service other tasks while the kernel is processing the 3D geometry.

Objects are defined with Model3D. Coordinates and normals are fixed-point integer data, while edge lists are byte index pairs.

Object Color Example Encoding

uint16_t object_color = 0x0D0B;  // index pair into bitmap palette
// far color  = 0x0D (high byte) (e.g. darker shade)
// near color = 0x0B (low byte) (e.g. lighter shade)

Hidden-Line Data Requirements

  • face_count > 0
  • face_nx/face_ny/face_nz arrays sized to face_count
  • edge_face0/edge_face1 arrays sized to edge_count
  • Use MODEL3D_INVALID_FACE (0xFF) for boundary/silhouette edges

5. Object And Scene Limits

Per Object Limits

  • VGK_MAX_VERTICES = 32
  • VGK_MAX_EDGES = 36
  • VGK_MAX_FACES = 16
  • VGK_MAX_INPUT_EDGES = 36

Slot/Scene Limits

  • 8 save slots (slot index 0..7)
  • VGK_SCENE_MAX_OBJECTS = 8
  • VGK_SCENE_MAX_VERTS = 256 combined
  • VGK_SCENE_MAX_EDGES = 288 combined
  • VGK_SCENE_MAX_CLIPS = 64 combined clip verts
Scene clip vertices share one global pool. Heavy clipping across many objects can exhaust VGK_SCENE_MAX_CLIPS before per-object edge/vertex limits are reached.

6. Demonstration Graphics Mode Setup

Geometry kernel output is edge data. The host must configure bitmap layers and a line rasterizer target before invoking draw routines.

Layer Configuration Pattern

  • video_init() sets up layer 0, layer 1, and layer 2 bitmap modes.
  • Layer 1 and layer 2 use CLUT 0 for 3D rendering (double buffer pair).
  • Layer 0 uses CLUT 1 for cockpit/overlay used with Anaconda demos.

Frame-Safe Drawing Sequence

  1. Pick hidden draw layer (draw_layer = visible == 1 ? 2 : 1).
  2. Clear hidden layer via dmaBitmapClear()
  3. Run geometry kernel and render edges into the bitmap layer using the appropriate function for single object or scene mode processing.
  4. Wait vblank via video_wait_vblank().
  5. Show new layer and hide old layer.
render_frame() in src/render.c is a production example of this double-buffer swap.

7. Kernel Initialization

// 1) Init display + plugin state
video_init();
vgk_plugin_init();
vs1053_dac_mute();   // Mute and disable DAC interrupts
vs1053_dac_interrupt_disable(); // Decoder RAM is being used.

// 2) Load model data into save slots
vgk_model_slot_init(&g_model_cube, 0);
vgk_model_slot_init(&g_model_anaconda, 1);

// 3) Projection defaults (example values)
vgk_projection_params_init(120, 160, 120, -64);

// 4) Camera defaults
vgk_cam_params_set(0, 0, 0, 0, 200, 1400);

// 5) Optional: service callback while waiting (example audio hook)
vgk_yield_cb_set(vgm_service_tick);

Model Load Variants

  • vgk_model_slot_init(model, slot) // vertices + edges + hidden-line payload
  • vgk_model_vertices_init(model, slot)
  • vgk_model_edges_init(model, slot)
  • vgk_model_hidden_line_init(model, slot)

Use split loads only if you intentionally manage partial slot updates.

8. Single Object Flow

Core Runtime Sequence

  1. Load target slot into active object context with vgk_model_load(slot).
  2. Set transform using vgk_obj_params_set(...).
  3. Reset/status clear if needed (vgk_reset()).
  4. Call vgk_trigger().
  5. Wait via vgk_wait_complete(timeout_ms).
  6. Read+emit edges with vgk_scrn_edges_with_depth_get(model, draw_layer).

Wait Return Codes

ReturnMeaningAction
1CompleteProceed to read output
2Kernel error statusInspect vgk_status(), recover/reset
0TimeoutTreat as stalled operation, retry/reset
Prefer vgk_scrn_edges_with_depth_get over vgk_scrn_edges_get; the depth version is optimized and integrates near/far color emission behavior.

9. Single Object Example

High-level example for rendering one spinning cube using the non-scene path. 

#define F256LIB_IMPLEMENTATION
#include "f256lib.h"

#include "../include/geometry_kernel.h"
#include "../include/3d_object.h"
#include "../include/video.h"

static uint8_t visible_layer = 1;
void app_init(void) {
    vs1053_clock_boost();
    vs1053_plugin_load();     
    vgk_plugin_init();
    vs1053_dac_mute();
    vs1053_dac_interrupt_disable(); // Decoder RAM is being used.  
    vgk_projection_params_init(240, 160, 120, -128);
    vgk_model_slot_init(&g_model_cube, 0);
    vgk_cam_params_set(0, 0, 0, 0, 200, 2400);
    vgk_hidden_line_enable();
    vgk_no_near_far_coloring = false;
}

void app_frame(uint8_t scale, uint8_t yaw, uint8_t pitch) {
    uint8_t draw_layer = (visible_layer == 1) ? 2 : 1;
    dmaBitmapClear(draw_layer);
    vgk_obj_params_set(pitch, yaw, 0, scale, 0, 0, -300);
    vgk_reset();
    vgk_trigger();
    uint8_t status = vgk_wait_complete(10000);
    if (status == 1) {
        vgk_scrn_edges_with_depth_get((Model3D *)&g_model_cube, draw_layer);
    } else if (status == 0){
        textPrint("Error: Geometry kernel timeout.\n");
    } 
    video_wait_vblank();
    bitmapSetVisible(draw_layer, true);
    bitmapSetVisible(visible_layer, false);
    visible_layer = draw_layer;
}


int main(int argc, char *argv[]) {
    (void)argc;
    (void)argv;
    uint8_t obj_yaw = 0;
    uint8_t obj_pitch = 0;

    video_init();
    app_init();
    textGotoXY(0, 0);
    textPrint("App Init Complete. \n");
    // make cube active object
    vgk_model_load(0);
    for (uint8_t loops = 0; loops < 4; loops++) {
        for(uint8_t frame=0; frame < 128; frame++) {
            app_frame(frame<64 ? 128 + frame : 192 - (frame - 64), obj_yaw++, obj_pitch++);
        }
    }
    textPrint("Done! Press any key to exit.\n");
    getchar();
    return 0;
}

10. Scene Mode

Scene mode processes multiple objects in one trigger using a descriptor array of SceneObjectParams records.

Scene Setup/Execution API

  • vgk_scene_enable(void)
  • vgk_scene_disable(void)
  • vgk_scene_set_descriptor(uint8_t n_objects, const SceneObjectParams *objects)
  • vgk_scene_set_object(uint8_t index, const SceneObjectParams *obj)
  • vgk_scene_set_object_count(uint8_t n_objects)
  • vgk_scene_get_result(SceneResult *result)
  • vgk_scene_object_meta_get(uint8_t index, SceneObjectMeta *meta)
  • vgk_scene_scrn_edges_get(...)

Occlusion and Ordering

  • Scene mode includes object-level depth ordering and AABB-based skip logic.
  • Disable scene occlusion sorting via vgk_scene_no_occlusion_enable() when needed.
  • Per-object hidden-line behavior still depends on each model's slot data.

Manual Scene Readback

For debug/analytics you can pull merged buffers directly:

  • vgk_scene_read_screen_coords(...)
  • vgk_scene_read_output_edges(...)
  • vgk_scene_read_clip_screen(...)

11. Multi-Object Example

High-level scene path using one-shot scene rendering helper. 

#define F256LIB_IMPLEMENTATION
#include "f256lib.h"

#include "../include/geometry_kernel.h"
#include "../include/3d_object.h"
#include "../include/video.h"

static SceneObjectParams scene_objs[8] = {
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x=-400, .pos_y=0, .pos_z=400 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x=-400, .pos_y= 0, .pos_z=-400 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x= 400, .pos_y=0, .pos_z=-400 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x= 400, .pos_y= 0, .pos_z=400 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x=-800, .pos_y=-0, .pos_z=800 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x=-800, .pos_y= 0, .pos_z=-800 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x= 800, .pos_y=0, .pos_z=-800 },
    { .slot=0, .yaw=0, .pitch=0, .roll=0, .scale=128, .pos_x= 800, .pos_y= 0, .pos_z=800 }
};

static uint8_t visible_layer = 1;
void app_init(void) {
    vs1053_clock_boost();
    vs1053_plugin_load();     
    vgk_plugin_init();
    vs1053_dac_mute();
    vs1053_dac_interrupt_disable(); // Decoder RAM is being used.  
    vgk_projection_params_init(240, 160, 120, -128);
    vgk_model_slot_init(&g_model_cube, 0);
    vgk_cam_params_set(0, 0, 0, 0, 200, 2400);
    vgk_hidden_line_enable();
}

void app_frame(uint16_t cam_x, int16_t cam_z, uint8_t cam_yaw) {
    uint8_t draw_layer = (visible_layer == 1) ? 2 : 1;
    dmaBitmapClear(draw_layer);
    vgk_cam_params_set(0, cam_yaw, 0, cam_x, 200, cam_z);
    vgk_reset();
    vgk_trigger();
    uint8_t status = vgk_wait_complete(10000);
    if (status == 1) {
        // retrieve and draw edges for the whole scene (all objects at once)
        vgk_scene_scrn_edges_get(8, scene_objs, 0x0B, 0x0D, draw_layer);
    } else if (status == 0){
        textPrint("Error: Geometry kernel timeout.\n");
    } 
    video_wait_vblank();
    bitmapSetVisible(draw_layer, true);
    bitmapSetVisible(visible_layer, false);
    visible_layer = draw_layer;
}

static uint8_t camera_orbit_angle;
static uint16_t camera_x;
static uint16_t camera_z;
static uint8_t camera_yaw;

static void camera_orbit(void) {

    const int16_t radius = 2400;
    const int16_t center_x = 0;
    const int16_t center_z = 0;

    camera_orbit_angle = (uint8_t)(camera_orbit_angle + 1u);
    int16_t sinv = sin_table[camera_orbit_angle];
    int16_t cosv = sin_table[(uint8_t)(camera_orbit_angle + 64u)];

    // Q14 multiply: (sinv * radius) >> 14
    int16_t dx = (int16_t)((mathSignedMultiply(cosv,radius)) >> 14);
    int16_t dz = (int16_t)((mathSignedMultiply(sinv,radius)) >> 14);

    camera_x = (int16_t)(center_x + dx);
    camera_z = (int16_t)(center_z + dz);

    // Face toward center: yaw = 64 - orbit_angle (sin/yaw mapping for this math setup)
    camera_yaw = (uint8_t)(64u - camera_orbit_angle);
}

int main(int argc, char *argv[]) {
    (void)argc;
    (void)argv;

    bool occlusion = true;
    video_init();
    app_init();
    textGotoXY(0, 0);
    textPrint("App Init Complete. \n");
    // use scene API for multi-object demo
    vgk_scene_enable();
    vgk_scene_set_descriptor(8, scene_objs);
    for (uint8_t loop = 0; loop < 4; loop++) {
        textGotoXY(0, 0);
        textPrint("Camera orbiting: Occlusion ");
        textPrint(occlusion ? "ON " : "OFF");

        if(!occlusion) {
            vgk_scene_no_occlusion_enable();
        } else {
            vgk_scene_no_occlusion_disable();
        }
        for(uint16_t frame=0; frame < 256; frame++) {
            if(frame % 2 == 0) {
                camera_orbit(); // updates camera_x, camera_z, camera_yaw
            }
            app_frame(camera_x, camera_z, camera_yaw);
        }
        occlusion = !occlusion;
    }
    textPrint("\nDone! Press any key to exit.\n");
    getchar();
    return 0;
}

14. Constants Reference

Key Limits

The plugin capability is constrained by the limited on-chip memory and the serial memory read/write process. The design currently limits single objects to 32 vertices, 36 edges and 16 faces. Of course many objects can be rendered to a frame by calling the plugin more than once or by using the scene feature.

  • VGK_MAX_VERTICES = 32
  • VGK_MAX_EDGES = 36
  • VGK_MAX_FACES = 16
  • VGK_SCENE_MAX_OBJECTS = 8
  • VGK_SCENE_MAX_VERTS = 256
  • VGK_SCENE_MAX_EDGES = 288
  • VGK_SCENE_MAX_CLIPS = 64

Trigger And Status

  • VGK_TRIGGER_MAGIC = 0xCAFE
  • VGK_STATUS_IDLE = 0
  • VGK_STATUS_BUSY = 1
  • VGK_STATUS_DONE = 0xABCD
  • VGK_STATUS_SAVE_ERROR = 0xE201
  • VGK_STATUS_LOAD_ERROR = 0xE202

Edge Flags

  • VGK_EDGE_VISIBLE = 0x0001
  • VGK_EDGE_CLIP_V0 = 0x0002
  • VGK_EDGE_CLIP_V1 = 0x0004
  • VGK_EDGE_NEAR = 0x0008
  • VGK_EDGE_CULLED = 0x0000
